# Размеры

У `DatePicker` и `DateRangePicker` есть проп `size` с тремя пресетами: `s`, `m`, `l`. По умолчанию — `m`.

Выбери пресет — справа сразу видно, как меняется высота, шрифт и отступы.

<SizePresets />

Тот же проп есть и у `DateRangePicker`:

```tsx
<DateRangePicker size="s" label="Период" />
```

## Что меняется

Размер — это набор CSS-токенов, которые подменяются модификатором-классом `.rtdp--s` / `.rtdp--l` (для `m` остаются базовые значения):

| Токен                       | `s`    | `m` (default) | `l`    |
| --------------------------- | ------ | ------------- | ------ |
| `--rtdp-height`             | `32px` | `40px`        | `56px` |
| `--rtdp-font-size`          | `14px` | `16px`        | `16px` |
| `--rtdp-padding-h`          | `12px` | `12px`        | `16px` |
| `--rtdp-input-padding-top`  | —      | `20px`        | `24px` |
| `--rtdp-label-top-filled`   | —      | `4px`         | `8px`  |

В размере `s` плавающий лейбл не используется — он зашит над полем (помещать его внутрь 32px невозможно без визуального компромисса).

## Кастомизация

Если хочется размер «между» — переопредели токены для нужного скоупа:

```css
.compact .rtdp {
  --rtdp-height: 36px;
  --rtdp-font-size: 15px;
  --rtdp-padding-h: 10px;
}
```

Это работает поверх любого `size` и не требует форка класса-модификатора.

## Попап календаря

Размер влияет и на класс попапа — `.rtdp__popover--s`, `.rtdp__popover--m`, `.rtdp__popover--l`. По умолчанию визуально это одинаковые попапы; модификатор есть на случай, если ты хочешь подогнать отступ/размер дней под высоту инпута:

```css
.rtdp__popover--s {
  --rdp-day-width: 32px;
  --rdp-day-height: 32px;
}
```
